Before submitting your pull request, please ensure your changes meet the following criteria:

- [ ] **No hardcoded passwords**  
  Confirm that no passwords or sensitive credentials are hardcoded in the code.

- [ ] **No personal data included**  
  Ensure the code does not include personal data such as home addresses, private emails, or any sensitive information.

- [ ] **Changes make sense for the project**

- [ ] **No merge conflicts**

- [ ] **Review feedback addressed**  
  Respond to all reviewer comments and make any required changes.

- [ ] **Ready to merge**  
  Verify the PR is ready for final review and merging.
